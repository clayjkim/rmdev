## 2026-06-13 (토)
- **'대응하기' 버튼 오류 긴급 수정**: 평판 화면에서 특정 글의 '대응하기'를 누르면 알 수 없는 영문 오류가 뜨며 저장이 안 되던 문제를 잡았다. 원인은 대응 기록을 저장하는 표(데이터베이스)에 '한 병원·한 글당 대응 하나'라는 **고유 규칙(제약)이 빠져 있어서**, 저장 시도가 매번 거부된 것 — 처음부터 빠져 있던 구조적 버그였다. 기존 데이터에 중복이 없음을 확인한 뒤 그 규칙을 추가해 **즉시 정상화**했다. 화면 코드는 멀쩡했고 데이터 쪽만 고친 거라 배포 없이 바로 적용.
- **'게시 내려감' 잘못된 표시 93건 바로잡고 원인 차단**: 멀쩡히 살아있는 글에 '게시 내려감(삭제됨)' 배지가 잘못 붙던 문제를 발견·수정했다. 원인은 글이 실제로 내려갔는지 매일 확인하는 자동 점검기가, 성예사·바비톡 페이지 **하단 약관에 흔히 들어있는 '게시중단'이라는 단어**를 보고 "이 글이 내려갔다"고 오판한 것 — 실제 페이지를 재현해 그 단어가 정확한 원인임을 확인했다. 그런 일반적인 단어들을 판정 기준에서 빼고(글 고유의 임시조치 통지문만 인정), 같은 페이지로 재검증해 '살아있음'으로 올바로 판정되는 것까지 확인. 잘못 붙은 배지 93건은 데이터에서 정리했다.
- **'대응하기' 누르면 확인 한 번**: 평판 글의 '대응하기' 버튼이 누르는 즉시 대응요청을 보내 오클릭 위험이 있었다. "대응요청 하시겠습니까?" 확인 단계를 한 번 넣어 실수 전송을 막았다.
- **휴대폰에서 데일리 리포트 PDF가 빈 화면으로 안 열리던 문제 수정**: 폰에서 데일리 리포트 PDF를 열면 내용이 안 보인다는 신고. **PDF 파일 자체는 멀쩡함을 확인**(직접 받아서 열어보니 검색노출 항목·통계·실제 검색화면 캡처까지 전부 정상)했고, 원인은 앱이 PDF를 '내려받기' 방식으로 주는데 그 방식이 **휴대폰(특히 아이폰)에서 빈 화면**이 되는 것이었다. 모바일에서는 PDF를 **브라우저 기본 뷰어로 바로 열도록** 바꿔 해결(PC는 기존 다운로드 유지).

## 2026-06-12 (금)
- **개발-배포 연결고리(깃허브↔화면도구) 2주 묵은 수수께끼 해결**: 5월 말부터 "코드를 깃허브에 올려도 화면 제작 도구(Lovable)에 반영되지 않는" 문제 때문에 모든 화면 수정을 프롬프트로만 전달하는 우회 방식을 써 왔다. 오늘 직접 실험(눈에 안 보이는 표식 심기, 12분 라이브 감시, 도구 화면 추적)으로 진상을 규명 — 5월 27일 밤 동기화 충돌 때 도구가 **새 저장소로 갈아탔고**, 우리 쪽(로컬)은 버려진 옛 저장소만 보고 있었던 것. 진짜 저장소로 전환하고, 옛 저장소에 갇혀 있던 문서 갱신분을 이식했으며, 잠들어 있던 동기화도 깨워 양방향 정상화. **이제 도구 크레딧을 쓰지 않고 코드를 직접 수정·반영**할 수 있어 개발 속도·비용이 크게 개선된다 (실서비스 공개만 버튼 1회 수동 — 안전장치로 유지).
- **재발 방지 수칙 제정**: 작업 전 저장소 신선도 검사 → 푸시 후 5분 내 반영 확인(무반응이면 가벼운 커밋으로 동기화 깨우기) → '재연결' 버튼 주의(누르면 새 저장소가 생겨 다시 어긋남) → 충돌 비상 브랜치가 나타나면 즉시 점검. 운영 기록에 명문화.
- **새 채널 — 구글 지도 리뷰(영문 대응 포함) 추가**: 병원 평판에서 구글 지도 리뷰는 빠질 수 없는데, 특히 외국인 환자가 많은 병원은 **영어·외국어 리뷰가 다수**(실측: 한 대형 병원 리뷰 1,199개 중 상당수가 영어). 구글은 공식 API가 리뷰를 최신 5개만 줘서 모니터링에 못 쓰고, 직접 수집은 차단이 강하다. 그래서 **무료 헤드리스 브라우저로 한국 IP에서 자동 수집**하는 방식을 채택 — 실제로 두 병원 리뷰를 긁어 구조·언어 비율을 검증한 뒤 본구축. 최신순으로 **새 리뷰만 증분 수집**(6시간 간격, 저빈도), 병원별 식별자만 넣으면 자동 편입되는 **멀티테넌트** 구조. 수집된 리뷰는 기존 분석 파이프라인이 그대로 받아 병원 매칭·위험도 분류까지 자동 처리됨을 확인.
- **외국어 리뷰 자동 번역·검수**: 기존 분류기는 한국어 전용이라 영어 리뷰를 놓칠 수 있었다. **외국어를 자동 감지해 AI(다국어)로 분류하고, 한국 홍보팀이 바로 읽도록 한국어 번역까지 동반**하도록 했다. 검증에서 영어로 된 심각한 불만("신경 손상·환불 거부·소송 언급")을 정확히 **'위험'으로 잡고 한국어로 요약**해냈다. 영어 부정 리뷰(외국인 환자 컴플레인)야말로 핵심 타깃인데 그걸 자동으로 검수하게 됐다.
- **집 미니PC(홈서버) 대비 이식형 설계**: 위 수집기는 경로 의존을 없애고 중복 실행 방지 장치를 넣어, 추후 집에 둘 **상시가동 미니PC로 폴더만 복사하면 그대로 24시간 가동**되도록 만들었다. 컴퓨터가 꺼져 있으면 30분 간격으로 재시도하고, 그래도 안 되면 유료 대행/클라우드로 넘기는 안전망까지 설계.
- **실시간 리뷰 목록에 '기간' 필터 추가**: '위험도순'으로 보면 위험한 글이 위로 오는데, 활동이 끊긴 **오래된 위험 글까지 계속 상단에 박혀** 있는 불편이 있었다. 화면에 **[24시간·3일·7일] 기간 버튼**을 두고 **기본을 '최근 3일'**로 맞춰, 최근에 새로 올라오거나 댓글이 달린(즉 지금 살아 있는) 글만 보이게 했다. **7일 넘게 잠잠한 글은 목록에서 빠진다** — '지금 대응이 필요한 것'에 집중되도록 정리. (기준은 글 작성일이 아니라 우리가 수집·감지한 시각이라, 오래된 글이라도 새 댓글이 달리면 다시 올라온다.)
- **소개(랜딩) 페이지 모바일 표시 정리**: 휴대폰에서 첫 화면의 큰 제목이 글자 중간에서 어색하게 잘려 보이던 문제를 바로잡았다('실시간으로'가 '실시간으'+'로'로 쪼개지던 현상). 한글이 **띄어쓰기 단위로만 줄바꿈**되도록 하고, 작은 화면에서는 제목 글자 크기가 자동으로 줄어 한 줄에 자연스럽게 담기도록 조정 — 소개 페이지의 모든 제목을 점검해 동일하게 적용했다.
- **모바일 앱 화면 메뉴 재설계**: 실제 사용이 휴대폰에서 많을 것을 감안해, 앱 하단 메뉴를 모바일에 맞게 정리했다. 기존에는 12개 항목이 하단에 옆으로 스크롤되며 끼어 있어 불편했는데, **하단에는 자주 쓰는 4개(실시간 리뷰·통계·인사이트·통화관리)만 큼직하게** 두고 나머지(채널 관리·대응관리·후기계약·설정·알림 설정)는 **'더보기'를 누르면 아래에서 올라오는 메뉴**로 묶었다. 대시보드는 위쪽 로고를 누르면 가도록 해 중복을 없앴다. 또 '데일리 리포트'와 '함께 언급 분석'은 인사이트의 하위 화면이므로, 별도 메뉴로 빼지 않고 **인사이트 안에서 탭으로 전환**하도록 했다. (PC 화면 메뉴는 그대로.)
- **모바일 통계·대시보드 숫자판 + 상담 버튼 정리**: 휴대폰에서 위쪽 요약 숫자판(전체·관련·위험·부정·긍정 리뷰 건수)이 한 줄에 2개씩 욱여넣어져 글자가 깨져 보였다. **휴대폰에서는 한 줄에 하나씩, 가로로 길고 높이가 낮은 띠 형태**(왼쪽 항목명·오른쪽 숫자와 증감)로 바꿔 시원하게 읽히도록 했다(PC는 기존 카드형 유지). 또 화면 오른쪽 아래에 떠 있던 **상담(채팅) 버튼이 하단 메뉴를 가리던 문제**는, 휴대폰에서는 그 버튼을 숨기고 **'더보기' 메뉴 안에 '상담하기'**를 넣어 해결했다(PC에서는 기존처럼 떠 있는 버튼 유지).
- **모바일 가독성 추가 정리(여백·아이콘·상세화면)**: ① 앱 화면의 좌우 바깥 여백을 휴대폰에서 최소로 줄여 내용 폭을 넓혔다. ② 리뷰 목록의 채널(플랫폼) 아이콘이 너무 커서 자리를 많이 차지하고 그 아래에 빈 공간이 생기던 것을, 휴대폰에서는 **작은 아이콘을 분류 뱃지 옆에 붙이고** 제목·본문이 가로 폭을 꽉 쓰도록 바꿨다. ③ 글 상세 화면이 바깥 여백과 박스 안쪽 여백이 이중으로 겹쳐 내용이 좁아 보이던 것을 줄여 시원하게 했다. ④ 실시간 리뷰의 기간 버튼(24시간·3일·7일)과 어긋나 보이던 '최근 3일 이내만 표시' 안내 문구를 없앴다. (PC 화면은 그대로.)

## 2026-06-11 (목)
- **리뷰 정렬 '위험도순' 추가**: 평판 화면의 글 목록은 기본이 '최신순'인데, 원장·홍보팀이 정작 보고 싶은 건 위험한 글이다. **[최신순 ↔ 위험도순] 토글**을 대시보드와 실시간 리뷰 양쪽에 넣어, 누르면 위험 → 부정 → 관심 → 긍정 → 무관 순으로 정렬되게 했다. 오래된 글이라도 위험하면 위로 올라온다.
- **멤버십 가격표 정비**: 한 요금제의 가격이 비어 있어 다른 요금제의 값이 잘못 표시되던 오류를 바로잡고, 규모별(1인 의원·중형·대형) 가격과 '전담 매니저 배정' 안내를 명시 — 가격을 보고 바로 이해되도록 정리.
- **대응 현황 화면 정리**: 기능을 보여주려 만들어 둔 빈 테스트 항목을 정리하고(실제 예시는 보존), '에스컬레이션'이라는 낯선 용어를 **'대응 중'**으로 바꿔 한눈에 들어오게 했다.
- **안내 문구 상시 노출 + 아이콘 고정**: "오래 전 글이라도 새 댓글·업데이트가 달리면 상단에 노출됩니다"라는 안내가 마우스를 올려야만 보이던 것을 항상 보이게 하고, 옆에서 깜빡이며 자리를 흔들던 새로고침 아이콘의 위치를 고정.
- **기간별 추이 그래프 수정**: '지난 3일·7일'을 고르면 전체 시장 그래프가 안 그려지던 문제를 해결(날짜 구간을 한국 시간 기준으로 맞춤). 이제 어느 기간을 골라도 일자별 추이가 정상 표시된다.
- **'함께 언급' 검색 속도 개선**: 분석량이 늘며 검색이 느려 결과가 안 뜨던 것을, 우리 고객 병원 범위로 좁혀 계산하도록 바꿔 응답을 절반 이하로 단축 — 이름 검색이 즉시 동작한다.

- **(4차) 대규모 데이터에서 대시보드가 느려지던 문제 — 근본 해결**: 수집 데이터가 19만 건을 넘으니 로그인·화면 진입이 매우 느려지고, 기간(지난 3일·한 달 등)을 고르면 숫자가 0으로 멈추는 현상. 직접 들어가 측정해보니 "전체 건수"를 셀 때마다 **19만 건을 통째로 훑어 21초**가 걸리고 있었다(데이터가 쌓일수록 더 느려지는 구조). **색인(데이터 찾아보기표)을 추가해 21초 → 0.3초로 단축.** 더불어, 화면의 "최근 리뷰"가 자꾸 비어 보이던 진짜 원인도 찾았다 — 수집한 글의 **작성일이 대부분 비어 있어**(일부 채널은 99% 이상), '작성일' 기준으로 거르면 최근에 수집한 글이 통째로 빠지던 것. 모니터링의 올바른 시간축인 **'우리가 감지한 시각'** 기준으로 바꾸도록 정리. 업계에서 흔히 쓰는 방식(집계는 서버에서, 화면은 가볍게, 색인·페이지 나눠 불러오기, 더 커지면 미리 합산해 둔 요약표)으로 방향도 세움.
- **(5차) 숫자와 목록이 어긋나던 문제**: '부정 3건'이라 적힌 카드를 누르면 18건이 뜨는 식으로, 카드 숫자와 클릭했을 때 목록 개수가 달랐다. 원인은 카드는 '기간+분류'로 정확히 집계하는데, 팝업은 미리 받아둔 일부 데이터를 분류로만 거르며 **기간을 무시**하던 것. "카드 숫자와 목록은 반드시 같은 조건으로 만든다"는 원칙으로 통일하도록 정리.
- **(6차) 인사이트 화면 정확도 — 경쟁사·항목 분석 바로잡기**: ① **자기 병원의 줄임말(초성 약칭)이 '경쟁사'로 잘못 잡히던 문제**를 근본 수정 — 글에서 자동으로 뽑은 이름들 중 자기 병원의 별칭·줄임말·변형과 의미 없는 자모 덩어리를 서버에서 걸러내도록 새로 만들었다. ② 시술 분류를 세분화하면서 남아 있던 **옛 분류('안티에이징')를 시장 트렌드에서 제거.** ③ 강점·약점 분석에서 **항목(예: 대기시간)을 클릭하면 실제로 그 내용이 담긴 글들이 바로 보이도록** 드릴다운 기능을 추가(사람이 분류가 맞는지 바로 확인 가능).
- **(7차) 대응 흐름 전면 정리(설계)**: 어느 화면(대시보드·통계·실시간 리뷰·인사이트)에서든 글마다 **[대응하기]와 [대응 불필요]** 버튼을 두고, '대응 불필요'를 누르면 대응 목록에서 빠지되 되돌릴 수 있게. 목록 화면들의 **표시 형식을 하나로 통일**(본문 미리보기·펼치기, 클릭이 안 되던 대응 버튼 정상화), 필터 기본값 정리(채널은 전체, 분류는 모두 켜짐), 대응관리 화면의 탭 순서·안내문 정리까지 한 묶음으로 설계해 적용 중.
- **(디자인) 전면 리디자인 — '다크 럭셔리' → '밝은 데이터 도구'**: 화면 톤을 금색·발광·둥근 모서리의 화려한 다크 테마에서, **밝은 본문 + 차분한 다크 헤더/사이드바 + 신뢰감 있는 인스티튜셔널 블루 + 직각의 정돈된 형태**로 바꾸는 작업. 병원 원장이 데이터를 '믿게' 만드는 시각 언어가 목표. 신뢰형 B2B·헬스케어·금융 대시보드를 근거로 디자인 원칙을 세우고, 토큰(색·모서리) 단위 기획안과 단계별 적용 프롬프트로 안전하게 전환 중.
